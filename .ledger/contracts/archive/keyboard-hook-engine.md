# 行为契约 — keyboard-hook-engine
版本：v1.0 | 关联 PID：.ledger/specs/keyboard-hook-engine-pid.md | 日期：2026-05-13

> 契约约束：键盘钩子回调中禁止执行 > 50ms 的操作（B-M04 异步回调风险缓解）。

## 功能契约（FC）

### 正常路径（关键路径，必须测试）
FC-01：当用户输入与某条已启用规则的 abbreviation 完全匹配的字符序列后，再输入终止键（Tab/Space/Enter 之一）时，系统删除已输入的缩写词字符，通过 SendInput 输出该规则的 replacement 文本，光标定位在替换文本末尾。多条规则匹配同一缩写词时，按 rules.json 数组顺序取第一条。
FC-02：当 replacement 文本包含动态变量时，系统在输出前解析：{date} → 当前日期（yyyy-MM-dd），{time} → 当前时间（HH:mm:ss），{clipboard} → 剪贴板当前文本内容。未识别的变量占位符原样输出。

### 边界与异常（关键路径，必须测试）
FC-03：当用户输入的字符序列不匹配任何已启用规则的 abbreviation 后按终止键时，系统不执行任何替换操作，终止键正常传递。
FC-04：当 rules.json 文件不存在时，系统使用空规则集启动，不报错。
FC-05：当 rules.json 文件内容为非法 JSON 时，系统忽略该文件使用空规则集，托盘气泡通知"配置文件异常，已忽略"。

### 次要异常（标注[可选]，不生成自动化测试）
FC-06：[可选] 当键盘钩子安装失败（权限不足或被安全拦截）时，系统通过托盘气泡通知"键盘钩子启动失败"。当 SendInput 调用失败时，系统静默失败，不执行替换，不报错。

## 非功能契约（NF）

### 可自动化测试
NF-01：当输入缓冲区长度超过 abbreviation 最大长度时，系统自动截断旧字符，保持缓冲区大小可控。
NF-02：当缩写词匹配成功并执行替换时，从匹配确认到 SendInput 完成的端到端耗时 < 100ms。

### 依赖环境（人工验收）
NF-03：键盘钩子在目标 Windows 环境（Win10/Win11）上能正常安装和触发，不被系统安全策略拦截。
NF-04：在中文输入法开启状态下，替换行为正常（不崩溃、不乱码，允许输入法状态下的已知局限）。

## 测试覆盖范围
自动化：FC-01 ~ FC-05，NF-01，NF-02
人工验收：FC-06（可选），NF-03，NF-04（依赖环境）

## 流程约束
- PAD 业务主流程 Step：S2
- 成功后用户去向：光标定位在替换文本末尾，继续正常输入
- 状态变化：HookState 从 Active 执行替换后回到 Active

## 体验一致性约束
- 错误提示：配置文件损坏时托盘气泡通知（继承 PAD 错误提示规则）
- 空状态：rules.json 不存在时静默使用空规则集（不打断用户）
- 重复操作：相同缩写词按配置顺序取第一条（继承 PID 失败场景）

## 明确不做
| 类别 | 内容 | 理由 |
|------|------|------|
| 永不做 | 运行外部动作 / 调用 API | 超出文本扩展工具边界 |
| 永不做 | 正则匹配 / 模糊匹配 | 缩写词精确匹配足够 |
| 本版本不做 | 进程绑定（不同软件不同规则） | MVP 不需要 |
| 本版本不做 | 剪贴板保护/恢复 | MVP 阶段简化 |
| 本版本不做 | 输入法状态检测和切换 | 复杂度高，收益低 |
| 简化做 | 多行文本不做特殊处理 | SendInput 逐字符发送，换行符自然传递 |

## 粒度说明

  - 跨模块影响：2 模块（中，2分）
