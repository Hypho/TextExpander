# PID Card — keyboard-hook-engine
版本：v1.0 | 状态：定稿 | 日期：2026-05-13

## 谁在使用？
Windows 用户（文职人员、开发者、方案架构师），在任意输入场景中需要快速输入预设文本。

## 他要做什么？
输入预设缩写词（如 ;addr）后按终止键（Tab/空格/回车），系统自动删除缩写词并替换为对应预设文本。

## 成功标准（正常路径）
- 输入 `;addr` + Tab → `;addr` 被删除，替换为预设地址文本
- 输入 `;date` + Tab → 替换为当前日期（如 2026-05-13）
- 输入 `;clip` + Tab → 替换为剪贴板当前内容
- 替换过程 < 100ms，用户无感知延迟
- 不影响未配置的正常输入

## 主流程映射
- PAD 业务主流程 Step：S2
- 功能类型：主流程
- 上游用户动作：S1 完成（规则已通过配置文件或 GUI 写入 JSON）
- 下游用户动作：继续当前工作（替换后无额外操作）
- 成功后用户去向：光标定位在替换文本末尾，继续正常输入

## 失败场景（边界与异常）
| 场景 | 触发条件 | 用户看到 |
|------|---------|---------|
| 钩子安装失败 | 权限不足或被安全软件拦截 | 托盘气泡通知"键盘钩子启动失败" |
| 配置文件缺失 | 首次运行无 rules.json | 使用空规则集，不报错 |
| 配置文件损坏 | JSON 格式错误 | 托盘气泡通知"配置文件异常，已忽略" |
| 缩写词冲突 | 多条规则匹配同一缩写词 | 取第一条匹配的规则（按配置顺序） |
| 替换文本含变量解析失败 | {date} 等变量格式错误 | 原样输出变量占位符 |
| SendInput 失败 | 目标窗口拒绝模拟输入 | 无动作，不报错（静默失败） |
| 键盘钩子回调超时 | 回调处理 > 300ms | Windows 自动移除钩子，托盘通知"钩子已断开，正在重连" |

## 明确不做
- 不做多行文本的特殊处理（SendInput 逐字符发送，换行符自然传递）
- 不做剪贴板保护/恢复（MVP 阶段简化）
- 不做输入法状态检测和切换
- 不做进程绑定（不同软件不同规则）
- 不做正则/模糊匹配
